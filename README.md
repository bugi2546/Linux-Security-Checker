# 프로젝트 이름 : Linux Server Security Diagnostic & Remediation Tool

---

##  프로젝트 간단 설명
주요정보통신기반시설 기술적 취약점 분석·평가 가이드라인(KISA)을 기반으로 리눅스 서버의 보안 취약점을 자동으로 진단하고 발견된 취약점을 안전하게 조치 및 복구하여 인프라 보안성을 강화하는 프로젝트입니다.

---

##  프로젝트 간단 요약
* **자동화 진단**: 쉘 스크립트(`check.sh`)를 통해 인프라의 주요 계정 보안 항목 5가지를 단 몇 초 만에 자동 진단
* **실시간 가시성**: 터미널 화면에 `[양호]`는 초록색, `[취약]`은 빨간색으로 알록달록하게 출력하여 시인성 극대화
* **증적 리포트 생성**: 진단 완료 시 대상 서버의 IP와 스캔 일시를 포함한 `Result_서버IP_날짜.txt` 파일이 자동 생성되어 영구 저장
* **실전 장애 대응**: 복잡한 PAM(Pluggable Authentication Modules) 인증 구조와 SSSD 환경을 분석하고 리커버리 모드(Single User Mode)를 통한 긴급 시스템 복구 기술 실증

---

##  진단 및 조치 보안 자원 (Target Items)
* **[U-01] root 계정 원격 접속 제한**: 외부에서 최고 관리자 권한으로 직접 무차별 대입 공격을 감행하는 경로 차단 `[양호]`
* **[U-02] 패스워드 복잡성 설정**: 시스템 사용자가 유추하기 쉬운 단순한 암호를 설정하지 못하도록 최소 규칙 강제 `[양호]`
* **[U-03] 계정 잠금 임계값 설정**: 5회 이상 로그인 실패 시 해당 계정을 10분간 자동 잠금하여 Brute Force 공격 원천 봉쇄 `[양호]`
* **[U-04] 패스워드 파일 보호**: 섀도우 패스워드(`shadow`) 체계를 적용하여 외부로의 비밀번호 해시값 유출 및 크래킹 방지 `[양호]`
* **[U-05] 패스워드 최소 길이 제한**: 시스템 전역 정책을 통해 최소 8자 이상의 강력한 패스워드 하한선 구축 `[양호]`

---

## 소프트웨어 / 하드웨어 요구사항
* **운영 체제**: Ubuntu Linux 20.04 LTS / 22.04 LTS 이상 환경 권한
* **가상화 플랫폼**: VMware Workstation Player / Pro 또는 VirtualBox
* **권한 요구사항**: 시스템 설정 파일(`/etc/`) 수정을 위한 최고 관리자(`sudo`) 권한 필수
* **필수 패키지**: Bash Shell, Nano Editor, OpenSSH-Server, PAM Modules

##  설치 및 실행 방법
1. 가상머신의 터미널 창을 실행(`Ctrl + Alt + T`)합니다.
2. 작성된 `check.sh` 스크립트 파일에 실행 권한을 부여합니다.
   ```bash
   chmod +x check.sh
3. 진단 스크랩트를 실행합니다.
   ```bash
   ./check.sh
